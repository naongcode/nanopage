# 나노바나나 기획 전달 프로세스

본 문서는 사용자 입력을 기반으로 기획안을 생성하고, 이를 "나노바나나"에 전달하는 전체 프로세스를 정의합니다.

## 프로세스 3단계 요약

1.  **[1단계] 사용자 입력:** 기획안 생성에 필요한 핵심 정보를 사용자가 입력합니다.
2.  **[2단계] LLM 기획 초안 작성:** 입력된 정보를 기반으로 LLM(대규모 언어 모델)이 기획안의 초안을 생성합니다.
3.  **[3단계] 수정 및 최종 전달:** 생성된 초안을 검토하고 수정한 뒤, 최종본을 "나노바나나"에 전달합니다.

## 단계별 상세 설명

### 1. 사용자 입력

사용자는 정의된 양식에 따라 기획안의 기초가 될 정보를 제공합니다. 이 단계는 결과물의 방향성을 결정하는 가장 중요한 과정입니다.

*   **입력 항목:**
    *   제품/서비스의 핵심 컨셉
    *   타겟 고객
    *   주요 기능
    *   마케팅 목표 등

### 2. LLM 기획 초안 작성

사용자로부터 받은 데이터를 사전 정의된 프롬프트 템플릿에 맞춰 구성한 뒤, LLM에 전달하여 기획안의 초안을 생성합니다.

*   **프로세스:**
    1.  사용자 입력 값 -> 프롬프트 변환
    2.  LLM API 호출 (기획안 초안 생성 요청)
    3.  생성된 텍스트(기획안 초안) 수신

### 3. 수정 및 최종 전달

LLM이 생성한 초안은 완벽하지 않을 수 있으므로, 내부 검토 및 수정 과정을 거칩니다. 오탈자, 논리적 오류, 비즈니스 요구사항과의 불일치 등을 확인하고 개선합니다.

*   **수정 사항:**
    *   문맥에 맞지 않는 표현 수정
    *   "나노바나나"의 요구사항 및 목표에 맞춰 내용 보강
    *   가독성 향상을 위한 형식 정리
*   **전달:**
    *   수정이 완료된 최종 기획안을 "나노바나나" 측에 전달합니다.
