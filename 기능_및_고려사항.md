# 기능 및 고려사항 (나노바나나 프로젝트)

본 문서는 "나노바나나" 프로젝트의 구현해야 할 주요 기능과 개발 및 운영 과정에서 고려해야 할 사항들을 정리합니다.

## 1. 구현해야 할 기능

### 1.1. 사용자 인터페이스 (UI)
*   **제품 정보 입력 UI:** 제품명, 카테고리, 경쟁사 컨셉, 타겟 고객 등을 입력받는 화면.
*   **마케팅 컨셉 입력 UI:** 핵심 셀링 포인트 3가지, 주요 시각적 컨셉, 톤앤매너, 필수 연출 상황 2가지, 금지 요소 1가지 등을 입력받는 화면.
*   **기획안 초안 검토/수정 UI:** LLM이 생성한 기획안 초안을 사용자가 확인하고 수정할 수 있는 에디터 기능.

### 1.2. 핵심 백엔드 로직
*   **프롬프트 생성 로직:** 사용자 입력을 기반으로 15장의 이미지 촬영 시나리오 프롬프트를 자동으로 생성하는 기능.
*   **프롬프트 변환 모듈:** 사용자 입력 값을 LLM 프롬프트 템플릿에 맞게 변환하는 기능.
*   **LLM API 연동 모듈:** 외부 LLM(대규모 언어 모델) API를 호출하고 응답을 처리하는 기능.

### 1.3. 결과물 처리 및 제공
*   **시나리오 결과물 제공:** 생성된 15개 시나리오를 역할 및 페이지 목적과 함께 표 형태로 시각화하여 제공.
*   **다운로드/복사 기능:** `.md`, `.txt` 형식으로 다운로드하거나 클립보드에 복사할 수 있는 기능.
*   **최종 기획안 전달 기능:** 수정 완료된 기획안을 "나노바나나" 측에 전달하는 기능 (예: 이메일 전송, 내부 시스템 연동 등).

## 2. 고려해야 할 사항

### 2.1. 사용자 경험 (UX)
*   **명확한 가이드라인:** 입력 필드별 명확한 설명과 예시를 제공하여 사용자가 정확한 정보를 입력할 수 있도록 유도.
*   **입력 편의성:** 복잡한 정보 입력을 최소화하고, 직관적인 UI를 통해 사용자 피로도 감소.

### 2.2. LLM 활용 전략
*   **프롬프트 엔지니어링:** LLM의 성능을 극대화하고 일관된 결과물을 얻기 위한 효과적인 프롬프트 템플릿 설계 및 지속적인 개선.
*   **초안의 품질:** LLM이 생성하는 기획안 초안의 품질을 높여 수정 작업의 효율성 확보.
*   **모델 선택:** 서비스 목적에 맞는 LLM 모델 선정 (예: 비용, 성능, 응답 속도 등 고려).

### 2.3. 시스템 아키텍처 및 확장성
*   **스토리텔링 구조 최적화:** 이커머스 상세페이지 특성을 반영한 15컷 스토리텔링 구조의 논리적 흐름 및 효과 검증.
*   **AI 프롬프트 구체화:** 배경/분위기, 감성/묘사, 기술적 연출 등 구체화 원칙이 프롬프트에 잘 반영되도록 설계.
*   **향후 확장성:**
    *   **AI 이미지 생성 연동:** Midjourney, DALL-E 등 외부 AI 이미지 생성 API 연동을 위한 설계.
    *   **스토리보드 기능:** 시나리오 순서 변경을 위한 드래그 앤 드롭 기능 구현 가능성 고려.
    *   **카테고리별 템플릿:** 다양한 제품 카테고리(의류, 식품 등)에 특화된 템플릿 관리 및 적용 방안.

### 2.4. 운영 및 유지보수
*   **내부 검토 프로세스:** LLM 초안 검토 및 수정에 대한 명확한 내부 가이드라인 및 워크플로우 수립.
*   **데이터 관리:** 사용자 입력 데이터 및 생성된 기획안 데이터의 저장, 관리, 보안 방안.
*   **성능 모니터링:** LLM 응답 시간, API 호출량 등 서비스 성능 모니터링 체계 구축.
